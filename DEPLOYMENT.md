# Deployment Guide - Render

This guide explains how to deploy the Worker Productivity Dashboard on Render.

## Prerequisites

- GitHub account with this repository pushed
- Render account (free tier works fine): https://render.com

## Quick Deploy (Recommended)

### Option 1: Using Blueprint (render.yaml)

1. **Push code to GitHub**:
   ```bash
   git add .
   git commit -m "Prepare for Render deployment"
   git push origin main
   ```

2. **Deploy on Render**:
   - Go to https://dashboard.render.com/
   - Click "New" → "Blueprint"
   - Connect your GitHub repository
   - Select this repository
   - Render will automatically detect `render.yaml` and create both services
   - Click "Apply"

3. **Update Frontend Environment**:
   - After backend deploys, copy its URL (e.g., `https://worker-productivity-backend.onrender.com`)
   - Go to Frontend service settings
   - Add environment variable:
     - Key: `VITE_API_URL`
     - Value: `https://worker-productivity-backend.onrender.com`
   - Save and trigger a manual deploy

4. **Access Your Application**:
   - Frontend: `https://worker-productivity-frontend.onrender.com`
   - Backend API: `https://worker-productivity-backend.onrender.com`
   - API Docs: `https://worker-productivity-backend.onrender.com/docs`

---

### Option 2: Manual Deployment

#### Deploy Backend

1. **Create Web Service**:
   - Go to Render Dashboard
   - Click "New" → "Web Service"
   - Connect your GitHub repository
   - Configure:
     - **Name**: `worker-productivity-backend`
     - **Runtime**: Python 3
     - **Build Command**: `pip install -r backend/requirements.txt`
     - **Start Command**: `bash backend/render_start.sh`
     - **Plan**: Free

2. **Add Persistent Disk** (Optional but recommended):
   - Go to service settings
   - Click "Disks" → "Add Disk"
   - Name: `backend-data`
   - Mount Path: `/opt/render/project/src/backend/data`
   - Size: 1 GB

3. **Environment Variables**:
   - `PYTHON_VERSION`: `3.11.0`
   - `PORT`: (auto-generated by Render)

4. **Deploy**:
   - Click "Create Web Service"
   - Wait for deployment to complete
   - Copy the service URL

#### Deploy Frontend

1. **Create Static Site**:
   - Click "New" → "Static Site"
   - Connect your GitHub repository
   - Configure:
     - **Name**: `worker-productivity-frontend`
     - **Build Command**: `cd frontend && npm install && npm run build`
     - **Publish Directory**: `frontend/dist`
     - **Plan**: Free

2. **Environment Variables**:
   - `VITE_API_URL`: `https://your-backend-url.onrender.com`

3. **Configure Routing** (Optional):
   - Add redirect rules to proxy API calls to backend

4. **Deploy**:
   - Click "Create Static Site"
   - Wait for deployment

## Important Notes

### Free Tier Limitations

Render's free tier has some limitations:
- **Services spin down after 15 minutes of inactivity**
- First request after inactivity may take 30-60 seconds (cold start)
- 750 hours/month free (sufficient for 1 always-on service)
- Web services share resources

### Database Persistence

- **With Disk**: Data persists across deploys and restarts
- **Without Disk**: Database resets on each deploy (data is lost)
- For production, consider using Render's PostgreSQL service instead of SQLite

### CORS Configuration

The backend is already configured to accept requests from any origin. For production, update the CORS settings in `backend/app/main.py`:

```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://worker-productivity-frontend.onrender.com"],  # Specific frontend URL
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

## Troubleshooting

### Backend Won't Start

**Issue**: ModuleNotFoundError or import errors

**Solution**: 
- Check that `requirements.txt` is in the `backend/` directory
- Verify build command: `pip install -r backend/requirements.txt`
- Verify start command: `bash backend/render_start.sh`

### Frontend Shows "Failed to load data"

**Issue**: Frontend can't connect to backend

**Solution**:
- Verify `VITE_API_URL` environment variable is set correctly
- Check backend URL is accessible: `https://your-backend.onrender.com`
- Rebuild frontend after setting environment variable

### Database Resets on Deploy

**Issue**: Data disappears after each deployment

**Solution**:
- Add a persistent disk to your backend service
- Mount path: `/opt/render/project/src/backend/data`
- Move database file to mounted path

### Cold Start Delays

**Issue**: First request takes 30-60 seconds

**Solution**:
- This is expected on free tier (services sleep after inactivity)
- Upgrade to paid tier for always-on services
- Use a health check service to ping your app periodically (e.g., UptimeRobot)

### CORS Errors

**Issue**: Browser blocks API requests

**Solution**:
- Verify CORS settings in `backend/app/main.py`
- Add your frontend URL to `allow_origins`
- Clear browser cache

## Upgrading to Production

### For Production Use, Consider:

1. **Upgrade to Paid Tier**:
   - No cold starts
   - More resources
   - Better performance

2. **Use PostgreSQL**:
   - Replace SQLite with Render PostgreSQL
   - Better for concurrent access
   - More reliable

3. **Add Redis**:
   - Cache metrics computation
   - Reduce database load
   - Faster response times

4. **Custom Domain**:
   - Add your own domain
   - Configure DNS
   - Enable HTTPS

5. **Monitoring & Alerts**:
   - Set up error tracking (Sentry)
   - Monitor uptime (UptimeRobot)
   - Configure alerts
